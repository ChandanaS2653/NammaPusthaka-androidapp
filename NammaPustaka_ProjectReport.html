2. Problem Statement

In many village schools, the library is just a cupboard of books with no tracking. Students often don't know what books are available, and teachers struggle to keep track of who has borrowed what. This leads to lost books and a lack of reading culture.

3. Vision

NammaPustaka is a "Smart Library Assistant" for rural schools. It turns a simple shelf into a digital catalog. Students can browse book covers on the app, see summaries, and "Reserve" a book. For the teacher, it acts as a digital register — sending reminders when a book is overdue.

4. App Features

📖 Book Catalog

Browse all books in a 2-column grid. Filter by category: Story, Science, History. Search by book name or author.

📷 QR Borrow

Scan a QR code on the book using the phone camera. Enter student ID to issue the book instantly.

📋 Active Borrows

View all currently borrowed books. Overdue items are automatically highlighted in red. Return books with one tap.

🏆 Reading Leaderboard

Ranks students by total pages read this month. Encourages healthy reading competition among students.

➕ Add New Book

Add books via a simple form. QR code is auto-generated. Cover color is randomly assigned. Camera-ready entry.

🌱 Seed Data

7 sample books and 4 students are pre-loaded on first launch for immediate testing without manual entry.

6. App Architecture

The app follows the MVVM (Model-View-ViewModel) pattern with a Repository layer separating data access from UI logic.

UI Layer (Fragment / Activity) │ ▼ ViewModel ←→ LiveData (observable data) │ ▼ Repository (single source of truth) │ ▼ Room DAO → SQLite Database

13. Key Implementation Details

QR Code Scanning

Uses CameraX 1.3.1 for live camera preview and ML Kit Barcode Scanning 17.2.0 for real-time QR code detection. A custom QRCodeAnalyzer implements ImageAnalysis.Analyzer and extracts the barcode value on each frame.

Overdue Detection

The DateUtils.isOverdue(dueDate: Long) function compares the due date timestamp with the current system time. The TransactionAdapter reads this flag and sets the text color to red (#F44336) and shows the "OVERDUE" tag.

Leaderboard Ranking

When a book is returned via TransactionRepository.returnBook(), the student's totalPagesRead is incremented by the book's totalPages. StudentDao.getAllStudentsRanked() queries with ORDER BY totalPagesRead DESC — the leaderboard auto-updates via LiveData.

Seed Data

Room's RoomDatabase.Callback.onCreate() inserts 7 books across Story, Science, and History categories plus 4 students on the very first app launch. This ensures the app is immediately usable without manual data entry.
